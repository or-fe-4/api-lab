# API Lab

FlaskでシンプルなREST APIを構築し、Nginxのリバースプロキシ経由で
公開した記録。

## 構成

ブラウザ -> Nginx(443番、SSL) -> Flask(5000番、内部のみ)

## やったこと

### Flask APIの構築
- GET / : API動作確認用
- GET /users : ユーザー一覧をJSON形式で返す
- GET /health : ヘルスチェック用エンドポイント

### Nginxとの連携(リバースプロキシ)
nginxの設定に/api/へのアクセスをFlask(localhost:5000)に転送する
location ブロックを追加。Flaskは5000番をAWSのセキュリティグループ・
ufwどちらにも公開せず、Nginx経由でのみアクセスできる構成にした。

### つまずいたポイント
EC2再起動でパブリックIPアドレスが変わっていたため、
古いIPでアクセスしてもつながらなかった。
curl ifconfig.meで現在の正しいIPを確認して解決。

## 学んだこと
- リバースプロキシはHA構成(Phase3)と同じ仕組みを別の用途に応用したもの
- アプリ(Flask)を直接公開せず、Nginxを窓口にすることでセキュリティを高められる
- インフラだけでなくアプリと組み合わせることで全体像が見えてくる
